About This Project

"We were so preoccupied with whether we could, we didn't stop to think if we should."

Why This Exists

There are plenty of nice apps for scanning paper documents to PDFs, and plenty of apps for rotating PDF pages. But very few nice apps let you drastically compress scanned PDFs with the highly efficient CCITT Group 4 or JBIG2 compression - which can shrink files by 95%+ while maintaining excellent clarity.

JBIG2 takes it further with lossy symbol matching: it finds similar-looking characters and reuses templates, achieving even better compression. At threshold 0.97, quality loss is imperceptible for most text documents.

Also, someone asked "can we make a single HTML file that contains its own source code as a compressed tarball?" and instead of saying "no, that's absurd," we said "hold my coffee."

Architecture: A Study in Questionable Decisions

The app works, but let's talk about the elephant in the HTML file: it contains its own source code as a base64-encoded tar.xz archive.

Why? Because when you're compressing PDFs, why not compress your own source code too? It's compression all the way down.

The file structure is essentially:

┌─────────────────────────────────────┐
│ HTML file (2.7 MB)                  │
│ ┌─────────────────────────────────┐ │
│ │ Decompression loader            │ │
│ │ (shows purple spinner)          │ │
│ └─────────────────────────────────┘ │
│ ┌─────────────────────────────────┐ │
│ │ gzip'd base64 application       │ │
│ │ ┌─────────────────────────────┐ │ │
│ │ │ The actual app              │ │ │
│ │ │ ┌─────────────────────────┐ │ │ │
│ │ │ │ tar.xz base64 source    │ │ │ │
│ │ │ │ (for rebuilding this)   │ │ │ │
│ │ │ └─────────────────────────┘ │ │ │
│ │ └─────────────────────────────┘ │ │
│ └─────────────────────────────────┘ │
└─────────────────────────────────────┘

It's like a turducken, but for web apps, and somehow smaller than the original version without the source code. Don't ask us how. We're still processing it ourselves.

What It Actually Does

Despite the architectural absurdity, this project actually works and is production-ready:

  • Converts any PDF to high-compression bilevel format
  • Renders pages at 310 DPI (configurable 72-1200)
  • Compresses with CCITT Group 4 (lossless) or JBIG2 (lossy with symbol matching)
  • JBIG2 via WebAssembly (jbig2enc compiled with Emscripten)
  • Outputs PDF/A-1B compliant files
  • Typical result: 10-80 KB per text page (CCITT G4), 5-50 KB (JBIG2)
