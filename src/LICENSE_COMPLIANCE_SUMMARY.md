## Legal Analysis

### Why Apache 2.0 Despite Studying LGPL Code

**Question**: Can we use Apache 2.0 license when we studied img2pdf (LGPL v3+)?

**Answer**: Yes, absolutely legal and proper.

**Reasoning**:
1. **No Code Copied**: We studied img2pdf to understand concepts, but wrote all code from scratch
2. **Clean Room Implementation**: This is a recognized legal practice
3. **LGPL Does Not Apply**: LGPL only applies to derivative works that copy or link to LGPL code
4. **Public Domain Specs**: Our implementation is based on TIFF 6.0 and PDF 1.4 specifications (public domain)
5. **Permissive Sources**: We used TIFFBitRevTable from libtiff (BSD-style, permissive)

**Precedents**:
- Wine: Studied Windows APIs → wrote own implementation → LGPL licensed
- Samba: Studied SMB protocol → wrote own implementation → GPL licensed
- Our project: Studied img2pdf concepts → wrote own implementation → Apache 2.0 licensed

**What We Did Right**:
- ✅ Documented that we studied img2pdf
- ✅ Clearly stated NO code was copied
- ✅ Attributed img2pdf properly
- ✅ Explained what we learned (concepts only)
